## Базовые настройки

### План счетов

Ничего не нужно делать

### Регистр бухгалтерии

- Указываем план счетов на вкладке `Основная`
- Ставим флажок `Корреспонденция` (нужно для формирования проводок вида дебет-кредит)
- Всегда добавляем ресурс `Сумма`. Устанавливаем флажок `Балансовый`


## Свойство, общее для всех проводок

Если есть разрез, который будет участвовать во всех проводках (например, организация), то создаем `измерение`


## Учет в валюте

### План счетов

Создаем **признак учета** `Валютный`

### Регистр бухгалтерии

Добавляем **небалансовый** ресурс `СуммаВалюта` и выставляем признак учета `Валютный`

### Примечания

- Чаще всего для учета по валюте будут использоваться счета, связанный с **контрагентами или агентами**. Выставляем этим счетам признак "Валютный"



## Простой учет остатков товаров

### План счетов

Создаем **признак учета** `Количесвенный`

### Регистр бухгалтерии

Добавляем **небалансовый** ресурс `Количество` и выставляем признак учета `Количесвенный`

### Примечания

- Чаще всего для учета количества будет использоваться счет "Товары", иногда в пару идут "Материалы" или "Товары в пути". Выставляем этим счетам признак `Количесвенный`
- Счетам, вроде "Прибыли убытки" неважна информация о количестве. Эти данные можно вытянуть из корреспондирующих счетов (например, из проводки со счетом "Товары")





## Остатки ведутся по какому-то признаку, а себестоимость - по всей компании

Подходит для ситуаций, когда остатки ведутся по одному признаку (например, склад), а расчет стоимости ведется в общем по компании.

Сначала нужно добавить настройки, как для [базовых настроек](#базовые-настройки).

### План счетов

У плана счетов создаем **признак учета субконто** `Суммовой`

Счет `Товары`:

|Субконто|Только обороты|Количественный|Суммовой|
|--------|--------------|--------------|--------|
|Номенклатура|			|+|+|
|Инвентарный номер|		|+| |

Счет `Прибыли убытки`: 
|Субконто|Только обороты|Количественный|Суммовой|
|--------|--------------|--------------|--------|
|Номенклатура|			|+|+|
|Инвентарный номер|		|+|+|

Для счета `Товары` обычно выставляется для:
- субконто `Номенклатура`

Для счета `Прибыли убытки` обычно выставляется для всех субконто.


### Регистр бухгалтерии

Для ресурса `Сумма` выставляем признак учета субконто `Суммовой`

### Примечания

Для запроса 
```1c
ВЫБРАТЬ
	// только номенклатура
	УправленческийОстатки.Субконто1 КАК Субконто1,
	УправленческийОстатки.КоличествоОстаток КАК КоличествоОстаток,
	УправленческийОстатки.СуммаОстаток КАК СуммаОстаток
ИЗ
	РегистрБухгалтерии.Управленческий.Остатки(, Счет = &СчетТовары, , ) КАК УправленческийОстатки
```

результат будет следующий:

Для запроса
```1c

	УправленческийОстатки.Субконто1 КАК Субконто1,
	УправленческийОстатки.Субконто2 КАК Субконто2,
	УправленческийОстатки.КоличествоОстаток КАК КоличествоОстаток,
```

## Остатки ведутся по одному признаку, а себестоимость рассчитывается по другому признаку

Подходит для случаев, когда остатки ведутся по одному признаку (например, склад), а списание и расчет себестоимости ведется по другому субконто (например, партии).


Сначала нужно добавить настройки, как для [базовых настроек](#базовые-настройки).

### План счетов

У плана счетов создаем **признак учета субконто**:
- `Количественный`
- `Суммовой`
- `Количественный<СубконтоОстатков>` (например, КоличественныйСклад)

Счет `Товары` будет выглядить следующим образом:

|Субконто|Только обороты|Количественный|КоличественныйСклад|Суммовой|
|--------|--------------|--------------|-------------------|--------|
|Номенклатура|			|+|+|+|
|Партия|				|+| |+|
|Склад|					| |+| |

Таким образом:
- `Номенклатура` получает все 3 флага
	- "КоличественныйСклад" - для контроля остатков
	- "Количественный" и "Суммовой" - для расчета себестоимости
- `Партия` получает 2 флага 
	- "Количественный" и "Суммовой" - для расчета себестоимости
- `Склад` полуает 1 флаг
	- "КоличественныйСклад" - для контроля остатков


### Регистр бухгалтерии

В регистр бухгалтерии добавляем ресурс `Количество<СубконтоОстатков>` (например, КоличествоСклад)
